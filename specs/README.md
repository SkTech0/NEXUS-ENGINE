# Specs

Single place for specifications, policies, and governance. All content here is documentation or config (YAML); no runtime code.

| Folder | Purpose |
|--------|---------|
| **contracts/** | API contracts (YAML) |
| **certification/** | Cert specs (consistency, determinism, performance, etc.) |
| **governance/** | Governance docs (data, lineage) |
| **gates/** | Readiness gates |
| **versioning/** | API/engine versioning, deprecation |
| **security/** | Security docs |
| **team-boundaries/** | Ownership, escalation, responsibility |
| **products/** | Product definitions (scope, vision, flows) |
| **api-platform/** | API governance, contracts, versioning |
| **market-readiness/** | Compliance, enterprise, technical |
| **release/** | Release strategy, pipelines, rollback |
| **ops/** | Runbooks, chaos, DR, incident response |
| **quality/** | Lint, coverage, performance, reliability rules (YAML) |
| **reports/** | Coverage, performance, security, SLA, test reports |

See [STRUCTURE.md](../STRUCTURE.md) for the full repo map.
